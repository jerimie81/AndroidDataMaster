**Strict Technical Constraints for this Workspace:**

1. **No Web Tech:** Do not generate any Node.js, React, or DOM-based code. This is a 100% native application.

2. **Architecture:** Android UI must be written in Java with XML layouts. All network requests, data parsing, and backend logic must be written in Rust compiled to a `.so` shared library.

3. **Security:** API keys must be injected into the Rust binary at compile-time via environment variables. Do not store keys in `strings.xml` or Java classes.

4. **Concurrency:** All JNI calls from Java must be executed on a background `ExecutorService` to prevent blocking the Android Main/UI thread. UI updates must be posted back via `MainLooper`.

**[CONTEXT: PROJECT ARCHITECTURE ROADMAP]**
We are refactoring a legacy web-based application into a highly performant, fully native Android application.

**1. Environment & Build Configuration:**

* Android project with a nested Rust library (`rust_core`).
* Android Build (`app/build.gradle`) uses a custom `Exec` task named `buildRust` to run `cargo ndk` for `armeabi-v7a`, `arm64-v8a`, `x86`, and `x86_64` targets, outputting directly to `src/main/jniLibs`.

**2. The Rust Core (`rust_core/`):**

* `crate-type = ["cdylib"]`.
* Uses `jni`, `reqwest` (blocking/json features), and `serde_json`.
* Exposes: `Java_com_jerimie_acrdai_MainActivity_processGeminiPrompt`.
* Logic: Formats a JSON payload for the Gemini 2.5 Flash API, executes a blocking POST request using the compiled-in API key, parses the response, and returns a `jstring` to Java.

**3. The Android UI (XML Layouts):**

* `activity_main.xml`: A `ConstraintLayout` with a `RecyclerView` for chat history and a bottom-pinned `LinearLayout` for the `EditText` and send button.
* `item_chat_message.xml`: A single layout for chat bubbles.
* `bg_chat_bubble.xml`: A 16dp rounded corner shape drawable.

**4. The Java Frontend (`com.jerimie.acrdai`):**

* `ChatMessage.java`: Data model containing text and an `isUser` boolean.
* `ChatAdapter.java`: Dynamically shifts bubble gravity (END for user, START for AI) and colors based on the sender.
* `MainActivity.java`: Manages the `RecyclerView`, loads the `rust_core` library, and handles the background threading for the JNI execution.
